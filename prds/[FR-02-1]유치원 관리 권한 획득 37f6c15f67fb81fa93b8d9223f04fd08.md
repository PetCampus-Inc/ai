## 1. 개요

### 목적

- 보호자 기본 계정 상태의 사용자가 특정 유치원의 원장 권한을 획득하고, 해당 유치원을 관리할 수 있는 권한과 매핑을 생성한다.

### 배경

- 기존 v2.0은 보호자의 유치원 탐색 경험 중심으로 운영되고 있어, 유치원 운영자가 직접 관리 기능을 사용할 수 있는 권한 구조가 필요하다.
- 1차 MVP에서는 운영 부담을 낮추기 위해 서류 업로드와 내부 수기 승인을 제외하고, 사업자등록번호 검증 기반의 즉시 권한 부여 구조로 시작한다.

### 대상 사용자

| 사용자 | 상태 | 목표 |
| --- | --- | --- |
| 잠재 원장 | 앱 가입 후 기본 보호자 권한을 가진 사용자 | 본인이 운영하는 유치원의 관리 권한을 획득한다. |
| 원장 | 유치원 관리 권한 획득 완료 사용자 | 유치원 관리 홈에 진입해 1차 MVP 운영 기능을 사용할 수 있다. |

### 성공 기준

- 사용자는 원장 인증 CTA에서 시작해 유치원 관리 권한 획득을 완료할 수 있다.
- 기존 DB 유치원 또는 신규 등록 유치원 중 하나에 사용자 계정이 원장으로 매핑된다.
- 권한 획득 완료 후 사용자는 유치원 관리 홈으로 이동할 수 있다.
- 중복 사업자번호, 이미 권한이 부여된 유치원, 휴업/폐업 사업자는 권한 획득이 차단된다.

## 5. 정책

| 정책 | 내용 |
| --- | --- |
| 기본 권한 | 회원 가입 직후 사용자의 기본 권한은 보호자이다. 원장 권한은 본 플로우 완료 후 추가된다. |
| 1계정 1유치원 | 1차 MVP에서는 한 계정이 하나의 유치원에 대해서만 원장 권한을 가질 수 있다. |
| 1유치원 1원장 선점 | 동일 사업자등록번호 또는 동일 유치원에 이미 승인된 원장 계정이 있으면 후속 신청은 차단한다. |
| 신규 유치원 즉시 활성화 | 기존 DB에 없는 유치원은 원장이 신규 등록할 수 있고, 제출 성공 시 즉시 ACTIVE 상태로 처리한다. |
| 인증 중 정보 수정 제한 | 기존 DB 정보가 일부 틀려도 인증 플로우 내에서는 수정하지 않는다. 인증 완료 후 유치원 정보 관리에서 수정한다. |
| 서류 심사 제외 | 사업자등록증, 동물위탁관리업 등록증 업로드와 내부 수기 심사는 1차 MVP 범위에서 제외한다. |
| 사후 권한 회수 가능성 | 허위 가입, 중복, 권한 분쟁, 폐업 유치원 등으로 확인되면 운영자가 원장 권한을 회수할 수 있다. 원장 탈퇴, 권한 종료, 권한 이전 절차는 [FR-02-5] 원장 탈퇴 및 유치원 운영 권한 종료를 따른다. |

| 기존 DB 정보 자동 입력 | Text | 유치원 정보는 인증 완료 후 관리 페이지에서 수정할 수 있어요. |
| 제출 완료 | 완료 화면 | 원장 인증이 완료되었어요. |
| 제출 실패 | Toast/Alert | 오류가 발생했어요. 네트워크 상태를 확인하고 다시 시도해 주세요. |
