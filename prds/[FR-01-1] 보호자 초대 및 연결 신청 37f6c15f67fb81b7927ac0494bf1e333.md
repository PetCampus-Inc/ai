## 1. 개요

### 목적

- 원장이 보호자를 유치원으로 초대하고, 보호자가 초대 링크를 통해 반려견 단위로 유치원 연결을 신청할 수 있게 한다.
- 보호자의 신청 경로를 초대 링크 중심으로 제한해, 1차 MVP에서 무분별한 연결 신청과 운영 혼선을 줄인다.

### 배경

- 1차 MVP에서 알림장, 등하원/출석, 원생 관리는 유치원과 보호자/반려견 간 연결이 선행되어야 사용할 수 있다.
- 원장이 단체 SNS, 메신저, OS 공유 기능을 활용해 보호자에게 초대 링크를 전달할 수 있어야 초기 전환이 가능하다.
- 기존 원본 PRD-FR-006은 초대, 신청, 승인/거절, 연결 해제까지 포함하고 있으나, 신규 구조에서는 초대/신청과 승인/상태 관리를 분리한다.

### 대상 사용자

| 사용자 | 상태 | 목표 |
| --- | --- | --- |
| 원장 | 유치원 관리 권한 획득 완료 | 보호자에게 유치원 연결 링크를 공유한다. |
| 보호자 | 초대 링크를 수신한 사용자 | 앱 설치/로그인 후 본인 정보와 반려견을 선택해 유치원 연결을 신청한다. |

### 성공 기준

- 원장은 보호자 초대 링크를 생성하고 공유할 수 있다.
- 보호자는 초대 링크를 통해 로그인/가입 후 유치원 연결 신청 화면으로 복귀할 수 있다.
- 보호자는 1마리 이상의 반려견을 선택하거나 생성해 유치원 연결을 신청할 수 있다.
- 보호자의 연결 신청은 원장 수동 승인 대기 상태로 저장된다.

## 5. 정책

| 정책 | 내용 |
| --- | --- |
| 초대 권한 | 1차 MVP에서는 유치원 관리 권한을 가진 원장만 보호자 초대를 생성할 수 있다. |
| 초대 경로 | 보호자 유치원 연결은 초대 링크 진입으로 제한한다. 앱 내 유치원 검색/상세에서 직접 연결 신청하는 경로는 제공하지 않는다. |
| 초대 링크 형식 | 초대 링크는 유치원을 식별할 수 있는 유치원별 고정 링크를 사용한다. |
| 초대 링크 유효기간 | 유치원별 보호자 초대 링크에는 만료 기간을 두지 않는다. |
| 초대 링크 공유 | 원장은 OS 기본 공유 시트를 통해 초대 링크를 보호자에게 전달한다. |
| 연결 단위 | 보호자-유치원 연결은 실제 운영 단위인 반려견/원생 단위 신청을 기준으로 처리한다. |
| 승인 대기 | 초대 링크로 제출된 신청은 원장 수동 승인 대기 상태로 저장한다. 승인 처리와 연결 상태 반영 상세는 다음 Feature에서 정의한다. |
| 재신청 | 이미 해당 유치원에 연결된 반려견은 중복 신청할 수 없다. 거절된 신청의 재신청 허용 범위는 연결 상태 관리 Feature에서 정의한다. |
