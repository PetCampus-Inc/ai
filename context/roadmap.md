> 이 문서는 장기 기능 목록이 아니라 **현재 가장 중요한 일을 잃지 않기 위한 우선순위 문서**다.

- 기준일: 2026-06-17
- 현재 단계: MVP
- 최상위 목표: 첫 실제 사용자 확보
- 핵심 KPI: 가입 수

## 2. 현재 우선순위 원칙

작업은 아래 순서로 우선한다.

1. 가입 자체를 막는 문제
2. 유치원 원장 또는 견주의 진입을 어렵게 하는 문제
3. 가입 후 첫 가치 경험을 막는 문제
4. 첫 사용자 확보를 위한 영업·초대·직접 온보딩 활동
5. 기존 핵심 흐름의 이해도와 신뢰를 높이는 개선
6. 사용자 근거가 있는 편의 기능
7. 미래 확장 기능
8. 꾸미기·자동화·고급 기능

## 3. NOW — 지금 해야 할 일

아래 항목은 확정된 기능 목록이 아니라, 현재 목표를 달성하기 위해 먼저 검토해야 하는 문제 영역이다.

### N1. 가입 KPI의 정확한 정의

**상태:** P0 후보  
**유형:** 측정·의사결정 기반

해야 할 일:

- 원장 가입 완료 정의
- 견주 가입 완료 정의
- 테스트 계정 제외 기준
- 가입 시작과 완료를 구분할 수 있는지 확인
- 유치원과 견주 중 어느 가입을 먼저 확보할지 결정

성공 기준:

- 팀 또는 운영자가 동일한 기준으로 가입 수를 셀 수 있다.
- 매주 가입 수 변화와 이탈 지점을 확인할 수 있다.

### N2. 현재 가입·초대 플로우 정리

**상태:** P0 후보  
**유형:** 사용자 흐름

해야 할 일:

- 원장 가입의 모든 단계를 문서화
- 견주 가입의 모든 단계를 문서화
- 유치원과 견주 연결 방식 확인
- 각 단계의 필수 입력과 실패 조건 확인
- 실제 기기에서 처음부터 끝까지 테스트

성공 기준:

- 한 장의 흐름도로 현재 상태를 설명할 수 있다.
- 가입을 막는 오류와 불필요한 단계를 구분할 수 있다.

### N3. 첫 사용자 후보 확보 및 직접 온보딩

**상태:** P0 후보  
**유형:** 사용자 확보

가설:

- 제품을 추가 개발하기 전에 실제 원장 또는 견주 한 명을 직접 가입시키는 것이 가장 빠른 학습 방법일 수 있다.

해야 할 일:

- 첫 유치원 또는 견주 후보를 정한다.
- 직접 가입 과정을 관찰한다.
- 막히는 지점과 질문을 기록한다.
- 필요하면 운영자가 수동 지원한다.

성공 기준:

- 실제 사용자 한 명이 가입을 완료한다.
- 가입 과정에서 발생한 문제 목록이 남는다.
- 다음 우선순위를 실제 관찰 근거로 결정한다.

### N4. 가입을 막는 가장 큰 장애물 하나 제거

**상태:** N2·N3 이후 결정  
**유형:** 제품 개선

규칙:

- 문제를 여러 개 동시에 고치지 않는다.
- 실제 사용자가 가장 크게 막힌 한 가지를 먼저 해결한다.
- 개선 전후 결과를 비교할 수 있어야 한다.

## 4. NEXT — 첫 가입 이후 검토할 일

첫 사용자 확보 후 실제 근거를 바탕으로 아래를 검토한다.

### A. 가입 완료율 개선

조건:

- 가입 시작은 있지만 완료가 적을 때

가능한 개선 예:

- 필수 입력 축소
- 가입 단계 순서 변경
- 인증 실패 안내 개선
- 초대 링크 복구
- 가입 후 다음 행동 안내

위 예시는 후보이며 실제 문제 확인 없이 구현하지 않는다.

### B. 첫 가치 경험 개선

조건:

- 가입은 하지만 핵심 행동을 하지 않을 때

확인할 것:

- 사용자가 가입 직후 해야 할 행동
- 원장과 견주 각각의 첫 성공 경험
- 빈 화면과 초기 데이터 없음 상태
- 샘플 또는 안내가 필요한지

### C. 반복 사용 여부 확인

조건:

- 첫 사용은 하지만 다시 방문하지 않을 때

확인할 것:

- 반복 방문 이유
- 알림 또는 콘텐츠의 실제 가치
- 원장과 견주 간 상호 의존성
- 운영 부담

## 5. LATER — 현재 미루는 영역

현재 목표에 직접 연결되지 않는 한 아래 작업은 후순위다.

- 고급 대시보드
- 정교한 통계
- 복잡한 자동화
- 대규모 조직 권한
- 다양한 요금제
- 추천 알고리즘
- AI 기능
- 커뮤니티 기능
- SNS형 기능
- 꾸미기와 프로필 장식
- 여러 국가·언어 대응
- 대규모 트래픽 최적화

이 목록은 영구 제외가 아니다.  
실제 가입 또는 사용을 막는 근거가 생기면 우선순위를 재평가한다.

## 6. 우선순위 등급

### P0 — 사용자 확보 또는 가입을 막는 치명적 문제

예:

- 가입 불가능
- 초대 링크가 작동하지 않음
- 인증 실패 후 복구 불가능
- 개인정보·법적 문제로 사용 불가능

### P1 — 첫 사용자를 확보하기 위해 지금 필요한 작업

예:

- 첫 가입 흐름 단순화
- 핵심 가치 경험에 필요한 최소 기능
- 직접 온보딩에서 반복 확인된 장애 제거

### P2 — 첫 사용자 이후 사용성을 높이는 작업

예:

- 반복 문의를 줄이는 안내
- 사용 빈도가 높은 흐름의 개선
- 활성화율 개선

### P3 — 근거가 부족하거나 나중에 해도 되는 작업

예:

- 사용자 요청 없이 제안된 편의 기능
- 미래 확장 대비
- 고급 설정

### P4 — 현재 목표에 맞지 않아 하지 않는 작업

예:

- 첫 사용자 확보를 방해할 정도로 큰 범위
- 검증할 수 없는 장기 프로젝트
- 문제 없이 기능만 늘리는 작업

## 8. 로드맵 변경 기준

다음 경우 우선순위를 다시 정한다.

- 실제 사용자가 가입 과정에서 막혔다.
- 가입 데이터가 기존 가설과 다르다.
- 첫 고객 후보가 명확한 요구를 제시했다.
- 법적·보안·개인정보 문제가 발견되었다.
- 작업 비용이 예상보다 크게 증가했다.
- 더 작은 범위로 같은 효과를 얻을 수 있다.
- 현재 1순위보다 가입에 더 직접적인 문제가 발견되었다.

## 9. 현재 최우선 권고

현재 확인된 정보만으로는 기능 개발을 바로 1순위로 확정하기 어렵다.

가장 먼저 할 일은 다음이다.

> **현재 원장·견주 가입 흐름을 문서화하고, 실제 사용자 후보 한 명을 직접 가입시켜 가장 큰 장애물을 관찰한다.**

이 권고의 근거:

- `[확정]` 현재 목표는 첫 사용자 확보다.
- `[확정]` 핵심 KPI는 가입 수다.
- `[확정]` 원장 또는 견주의 접근이 어려운 것이 주요 우려다.
- `[미확인]` 실제로 어느 단계에서 이탈하는지는 모른다.

따라서 관찰 없이 기능부터 추가하면 잘못된 문제를 해결할 위험이 크다.
